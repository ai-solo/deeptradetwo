## **项目核心架构分析**
DeepTrade 是一个高度工程化的 **AI 驱动型 ETH 期货量化交易 Agent**。其核心逻辑在于将海量的实时市场数据（价格、指标、深度、资金费、交易流等）转化为 LLM 可理解的结构化提示词，利用大模型的推理能力进行决策。

### **1. 模块化设计**
- **[binance](file:///Users/zhangyang/Desktop/freedom/deeptrade/binance)**: 封装了与币安交易所的交互，支持期货 API、多环境切换（测试网/生产网）。
- **[indicators](file:///Users/zhangyang/Desktop/freedom/deeptrade/indicators)**: 独立的技术指标库，包含从基础均线到复杂的交易流分析（Trade Flow）和成交量分层分析（Volume Layers）。
- **[task](file:///Users/zhangyang/Desktop/freedom/deeptrade/task)**: 业务逻辑核心，负责数据聚合 (`market_data.go`)、指标格式化 (`technical_analysis.go`)、LLM 分析 (`analysis.go`) 和交易执行 (`execution.go`)。
- **[utils](file:///Users/zhangyang/Desktop/freedom/deeptrade/utils)**: 通用工具类，集成了 LLM 调用框架（基于 Cloudwego Eino）和邮件报警系统。

### **2. 核心交易流程**
1. **数据采集**: 每 20 分钟（可配置）触发一次，并发获取 K 线、深度、持仓、资金费等全维度数据。
2. **特征工程**: 将原始数据转化为技术指标和趋势描述（如“24h 区间位置”、“EMA 队列状态”）。
3. **AI 决策**: 
   - **开仓**: 使用高性能推理模型（如 DeepSeek Reasoner）进行深度市场分析。
   - **持仓管理**: 使用响应速度更快的模型（如 Qwen Plus）进行实时风险跟踪。
4. **执行与记忆**: 自动下达市价或限价单，并更新 `memory` 字段，确保决策的连续性。

## **技术亮点**
- **COT (Chain of Thought) 决策模式**: 系统提示词设计极其精良，要求模型进行“信号一致性检查”和“风险量化”，将专业交易员的思维逻辑植入 AI。
- **状态记忆系统**: 通过 LLM 的 `memory` 字段实现了轻量级的状态机，使 Agent 能够“记住”上一次的判断逻辑，有效避免在震荡行情中反复“打脸”。
- **精细化的风险控制**: 内置基于 ATR 的动态止损止盈逻辑，并有严格的交易时间窗限制（避开非活跃时段）。
- **多维度数据融合**: 结合了微观结构（订单簿深度）和宏观趋势（K 线指标），比单一维度的机器人更具鲁棒性。

## **优化建议**
1. **回测系统 (Backtesting)**: 目前系统侧重于实盘/模拟盘执行，建议增加基于历史数据的回测模块，用于验证 LLM 策略在极端行情下的表现。
2. **多币种支持**: 目前专注于 ETH，可以考虑抽象出币种配置，实现多品种并行交易。
3. **持久化存储**: 目前的交易记录和记忆主要在内存或日志中，建议引入数据库（如 SQLite 或 PostgreSQL）存储详细的决策链路，便于后续分析和模型微调。
4. **多模型投票机制**: 引入多个不同厂商的 LLM 进行信号共识校验，进一步降低单点模型幻觉带来的风险。
